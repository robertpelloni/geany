# Project Vision: The Ultimate IDE

## Ultimate Goal
The ultimate goal of this project is to take the Geany IDE and elevate it into an absolutely outstanding, magnificent, extraordinary, phenomenal, marvelous, INSANELY GREAT text editor and IDE.

## Core Pillars
1. **Clean Architecture & Language**: Refactor the existing C codebase to clean, well-structured, well-thought-out, well-named, well-organized, straightforward, easy-to-use modern C++.
2. **The Go Ultra-Project**: Concurrently port the entire project to Go (`geany-go`). Every feature and functionality possible must be completely ported to Go in a robust and superior implementation. All submodule projects will be analyzed, and their features assimilated and implemented until all projects are merged into one massive, streamlined Go ultra-project.
3. **1:1 Feature Parity with Notepad++**: Deeply analyze Notepad++ (a competing but inferior text editor) and ensure Geany has 100% 1:1 feature parity and functionality parity. Every detail, feature, menu item, button, option, and setting must have an analogous feature in Geany, improved wherever possible.
4. **Multiple Native UI Frontends**: The project will utilize submodules to build multiple native UI frontends:
   - `bobui` (bobq / qt6)
   - `btk` (copperspice / qt4)
   - `bobgui` (bobtk / gtk)
   - A Web UI (planned for the future).
5. **Comprehensive Documentation**: Every single implemented and planned feature must be well represented in full detail in the UI (labels, descriptions, tooltips) and fully documented in manuals, help files, and roadmap/todo lists.

## The "Never Stop" Philosophy
Development should be autonomous, continuous, and highly rigorous. Every feature must be thoroughly tested, documented, and refined.
